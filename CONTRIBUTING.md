# Contributing to mv3-migrate

We welcome contributions. Here is how to get started.

QUICK START

1. Fork the repo
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/mv3-migrate.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature`
5. Make your changes
6. Run tests: `npm test`
7. Run the linter: `npm run lint`
8. Commit with a clear message: `git commit -m "feat: your feature description"`
9. Push: `git push origin feature/your-feature`
10. Open a Pull Request

GUIDELINES

- Write TypeScript with no `any` types
- Add tests for new features
- Follow existing code style
- Keep commits atomic and descriptive
- Update the README if you add new features or change the CLI interface

REPORTING ISSUES

- Use the issue templates for bugs and feature requests
- Search existing issues before creating new ones
- Include reproduction steps for bugs, including your Node.js version and OS

CODE OF CONDUCT

Be respectful and constructive. We are all here to build great tools.
